Recordatorio de que las normas de nomenclatura son importantes para identificar facilmente los elementos.
La consistencia y facilidad son esenciales para mantener y expandir el codigo correctamente.

Util:
	- Usar nombres adecuados para funciones, variables....
	- Sin miedo a renombrar, ya que es para que sea mejor.
	- Nunca usar letras simples, acronimos o abreviaturas, si hay bug, 
	  sera mas facil de recordar que hace ese codigo ejemplo (p => pos => position)
